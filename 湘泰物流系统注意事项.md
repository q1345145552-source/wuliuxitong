## 三、三端说明

| 端 | 角色 | 权限 |
|------|------|------|
| 客户端 | client | 看自己运单、充值、付款、上传凭证 |
| 员工端 | staff | 全部运单管理、创建订单、分柜、尾端派送 |
| 管理员端 | admin | 全部权限 + 删除运单、审核充值/付款 |

## 四、已实现功能

### 运单查询
- 客户端：localStorage 缓存 → 刷新秒显示 → 10秒后台自动刷新
- 展开详情面板：基本信息/财务信息/产品明细/产品图片

### 充值功能
- 客户端：余额页面 → 选币种/金额/支付方式/上传凭证 → 提交
- 管理员：充值审核 → 通过（余额增加）/拒绝（填原因）
- 员工：客户余额只读查看
- 汇率：每天0点自动更新 CNY/THB

### 付款功能
- 余额支付：选运单 → 余额扣款 → 即时到账
- 线下支付：上传凭证 → 管理员审核 → 通过/拒绝
- 管理员：付款审核选项卡

### 装柜管理
- 路径：`/staff/container-loading`
- 管理员和员工均可访问

## 五、关键规则

- **客户端绝对不能看柜号、船次等装柜信息**
- **所有显示用运单号，不用内部 ID（o_xxx）**
- **修改代码后必须先 npm run build 再 push**
- **Order 表没有 trackingNo 字段，trackingNo 在 Shipment 表**
- **Token 有效期 7 天，过期自动跳登录页**

## 六、常见问题

| 问题 | 解决 |
|------|------|
| 运单查询慢 | localStorage 缓存 + 10秒轮询 |
| missing auth context | token 过期，退出重新登录 |
| 登录页死循环 | ?expired=1 自动清除旧 session |
| API 500 错误 | 检查是否 select 了 Order 表中不存在的 trackingNo |
| Docker 构建缓存 | `docker compose build --no-cache api` |
